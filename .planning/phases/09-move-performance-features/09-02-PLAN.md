  <files>public/engine/capture.js, public/engine/instruments.js</files>
  <action>
Create `public/engine/capture.js`:

**Rolling note buffer** — continuously records recent note events with timestamps, keeping the last 1 measure (16 steps) worth of data.

State:
- `buffer` (array of `{ note, velocity, timestamp }` entries)
- `bufferDurationMs` — dynamically calculated as `(60000 / getBPM()) * 4` (one measure = 4 beats at current BPM). Recalculated on each feedCapture call.

Imports: `getBPM, isPlaying` from `./sequencer.js`, `getActiveTrack` from `./track-manager.js`, `NUM_STEPS` from `./melodic-sequencer.js`, `setSelectedNote` from `./melodic-sequencer.js`.

**Exports:**

`feedCapture(note, velocity)` — called from instruments.js noteOn. Adds `{ note, velocity, timestamp: performance.now() }` to buffer. Prunes entries older than bufferDurationMs from the front. This runs on every noteOn regardless of whether user will capture — it's lightweight (push + shift).

`commitCapture()` — quantizes the buffer contents into the active track's step grid:
1. Get `activeTrack = getActiveTrack()`. If not melodic type, return early.
2. Get current BPM and calculate step duration: `stepDurationMs = (60000 / getBPM()) / 4` (each 16th note step).
3. Determine the time window: `windowEnd = performance.now()`, `windowStart = windowEnd - (stepDurationMs * NUM_STEPS)`.
4. For each buffer entry within the window:
   a. Calculate which step it falls on: `step = Math.floor((entry.timestamp - windowStart) / stepDurationMs)`.
   b. Clamp step to 0..15.
   c. Add the note to `activeTrack.grid[step]` (it's a Set, so duplicates are safe).
5. Set the last captured note as the selected note via `setSelectedNote(lastNote)` so the step display shows it.
6. Dispatch `'melodic-update'` CustomEvent so step-buttons refresh.
7. Clear the buffer after commit.
8. Return the count of notes committed (for UI feedback).

`clearCaptureBuffer()` — empties the buffer. Called when switching tracks or manually.

**Integration in instruments.js:**
- Import `feedCapture` from `./capture.js`.
- In `noteOn()`, after the existing synth trigger and recording check, add: `feedCapture(note, velocity);`
- This goes AFTER the arp check from Plan 01. If arp is enabled, the arp tick calls noteOn internally, which will still feed capture. This is correct behavior — capture should record what actually sounds.

Note: feedCapture should be called unconditionally (even when not "in capture mode") because the whole point is retroactive — the buffer is always rolling. The commit is the conscious action.
