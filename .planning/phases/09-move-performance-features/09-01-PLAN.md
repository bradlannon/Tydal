Create `public/engine/arpeggiator.js`:

**Arpeggiator engine** — BPM-synced note cycling through held notes in Up, Down, or Random order.

State:
- `enabled` (boolean, default false)
- `mode` ('up' | 'down' | 'random', default 'up')
- `rate` ('8n' default — same subdivision options as note-repeat: '4n', '8n', '16n', '32n')
- `heldNotes` (array of note strings in the order they were pressed)
- `arpIndex` (current position in the sorted/shuffled note array)
- `intervalId` (setInterval handle for the arp tick)
- `lastNote` (the last note triggered, for noteOff before next)

Imports: `noteOn`, `noteOff` from `./instruments.js`, `getBPM` from `./sequencer.js`

Rate-to-ms conversion: same pattern as note-repeat.js — `rateToMs(noteRate)` using `60000 / getBPM()` quarter note base.

**Exports:**
- `addArpNote(note, velocity)` — called from pad-grid touch handler when ARP is enabled. Adds note to heldNotes (if not already present). If this is the first note and arp is enabled, starts the arp interval. Does NOT call noteOn directly — the arp tick handles it.
- `removeArpNote(note)` — removes note from heldNotes. If heldNotes becomes empty, stops interval and calls noteOff on lastNote.
- `setArpEnabled(val)` — enable/disable. Disabling stops interval and calls noteOff on lastNote.
- `isArpEnabled()` — returns boolean.
- `setArpMode(mode)` — sets 'up'/'down'/'random'. Takes effect on next tick cycle boundary.
- `getArpMode()` — returns current mode string.
- `setArpRate(rate)` — sets subdivision. Restarts interval immediately if running (same pattern as note-repeat setRepeatRate).

**Arp tick logic:**
1. If heldNotes is empty, stop interval and return.
2. Sort heldNotes by MIDI note number for Up/Down modes. Use Tone.Frequency(note).toMidi() or a simple sort by note name. For 'random', shuffle.
3. Call noteOff on lastNote (with 5ms gap via setTimeout before noteOn, matching note-repeat pattern).
4. Determine next note based on mode:
   - Up: increment arpIndex mod sorted.length
   - Down: decrement arpIndex (wrap to end), or reverse-sort and increment
   - Random: pick random index from sorted array
5. Call noteOn(nextNote, 0.7).
6. Store nextNote as lastNote.

**Swing in sequencer.js:**
- Add module-level `let swingAmount = 0;` (0-1 range, 0 = straight, 1 = full triplet swing).
- Export `setSwing(amount)` and `getSwing()`.
- In the Tone.Sequence callback, apply swing timing to odd steps (1, 3, 5, 7, 9, 11, 13, 15). Swing delays odd-numbered 16th notes. The standard technique: for odd steps, add a time offset of `swingAmount * (60 / getBPM() / 4)` seconds (which is one 16th note duration times swing amount). This means at swing=0 no offset, at swing=1 the off-beat is delayed by a full 16th note (making it a triplet-ish feel). Use `time + offset` when calling triggerKick/triggerSnare/etc for odd steps. Even steps play on time.
- The scheduleStepEvent visual dispatch should NOT be swung — visual cursor stays on-grid for clarity.

**Swing in melodic-sequencer.js:**
- Import `getSwing, getBPM` (getBPM already available via sequencer.js or add import).
- Apply same swing offset logic in the melodic Sequence callback for odd steps. Pass `time + offset` to `track.synth.triggerAttackRelease()`.
- Import getSwing from sequencer.js so there is a single swing source of truth.
