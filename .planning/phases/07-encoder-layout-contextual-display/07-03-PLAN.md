  <name>Task 1: Create jog wheel component for preset browsing</name>
  <files>public/ui/jog-wheel.js, public/styles.css</files>
  <action>
Create `public/ui/jog-wheel.js`:

- `initJogWheel(containerEl, oledEl)` — builds a jog wheel element and appends to containerEl. Uses the shared OLED display element (passed in) to show preset names
- The jog wheel is a circular element, larger than encoders: 56px diameter on mobile, 64px on desktop
- Visual: `#111` background, `#333` border (2px), border-radius: 50%. A subtle radial groove pattern (CSS gradient: `conic-gradient(from 0deg, #111, #1a1a1a, #111)` or similar) to suggest rotation. Center dot: 6px, #333
- Touch gesture: same vertical drag as encoders, but each "notch" (20px of drag) moves to the next/previous preset in the list. Use pointer capture
- During drag: each notch fires a preset change. Show the current preset name on the OLED display via `showOLED(oledEl, 'Preset', presetName)`. Do NOT apply the preset until the user releases (commits)
- On pointerup (commit): apply the selected preset via `applyPreset(name)`. Hide OLED after standard delay
- On pointerdown: show current preset name immediately on OLED
- Internal state: maintain a `currentIndex` into `getPresetNames()` array. Start at 0 (first preset). Wrap around at boundaries (circular scrolling)
- The jog wheel also handles browsing when the DRM toolbar button is active — in drum mode, it scrolls through drum kit names. For Phase 7, there's only one drum kit, so in drum mode the jog wheel shows "808 Kit" and does nothing on scroll. Phase 8 will add multi-kit support.
- Export: `initJogWheel(containerEl, oledEl)`

**CSS additions to styles.css:**
- `.jog-wheel` — width: 56px, height: 56px, border-radius: 50%, background: #111, border: 2px solid #333, cursor: grab, touch-action: none, position: relative, flex-shrink: 0, -webkit-tap-highlight-color: transparent
- `.jog-wheel-inner` — position: absolute, inset: 8px, border-radius: 50%, background: conic-gradient(from 0deg, #151515 0%, #1a1a1a 25%, #151515 50%, #1a1a1a 75%, #151515 100%), border: 1px solid #222
- `.jog-wheel-center` — position: absolute, top: 50%, left: 50%, transform: translate(-50%, -50%), width: 6px, height: 6px, border-radius: 50%, background: #444
- `.jog-wheel.active` — border-color: #555
- `.jog-wheel.active .jog-wheel-center` — background: #fff
- Desktop media query (min-width: 768px): `.jog-wheel` width/height: 64px

**Jog wheel placement:**
- The jog wheel sits to the RIGHT of the encoder row. In index.html, the `#encoder-section` div already exists (from Plan 01). The encoder-row.js module creates the OLED + encoder row inside it
- Modify the encoder section layout: OLED display spans the full width on top, below that a flex row containing the 9 encoders on the left and the jog wheel on the right
- In `initEncoderRow()` (encoder-row.js), after creating the encoder row, accept an optional callback or element for the jog wheel area. OR: handle jog wheel initialization in app.js by getting the OLED element reference from encoder-row.js

**Approach — keep it simple:**
- In app.js: after `initEncoderRow(encoderSectionEl)`, get the OLED element reference (encoder-row.js should export `getOLEDElement()` or attach it as a property)
- Add a jog wheel container div adjacent to the encoder row (inside encoder-section), and call `initJogWheel(jogContainer, oledEl)`
- Update encoder-row.js to export `getOLEDElement()` returning the OLED display element

**Contextual encoder mapping (MPERF-05):**
- In encoder-row.js, flesh out the `DRUM_MAPPING` array with actual drum parameters:
  1. Kick Tone — placeholder (no direct param exposed yet; use drumBus.volume as temp)
  2. Snare Tone — placeholder
  3. HH Decay — placeholder
  4. Clap Verb — placeholder
  5. Reverb Wet → reverb.wet.value (drums still route through shared reverb for now)
  6. Delay Wet → delay.wet.value
  7. BPM (40–240, default 120) → setBPM(val); updates Transport
  8. Drum Vol (-40–0, default 0) → drumBus.volume.rampTo(val, 0.05)
  9. Master Vol (-40–0, default -6) → masterVolume.volume.rampTo(val, 0.05)
  - Note: Placeholders (encoders 1-4) show the name but onChange is a no-op. This is intentional — individual drum voice parameters are module-private in drums.js. Phase 8's multi-track refactor will expose per-voice controls. For now, 5 of 9 drum encoders are functional.

- `setEncoderMapping(mappingArray)` — updates all 9 encoder configs: sets new min/max/value/step/onChange on each encoder, resets their visual positions, updates internal mapping reference
- Add a listener for toolbar button state: when DRM toolbar button is activated (toolbar btn has `.active` class and `data-sheet="seq-sheet"`), call `setEncoderMapping(DRUM_MAPPING)`. When any other toolbar button is activated or DRM is deactivated, call `setEncoderMapping(MELODIC_MAPPING)`
- Implementation: listen for `click` events on toolbar buttons (`document.querySelectorAll('.toolbar-btn[data-sheet]')`) and check which one is active to decide mapping
- OR: simpler approach — dispatch a CustomEvent `mode-change` with detail `{ mode: 'drum' | 'melodic' }` from app.js when toolbar buttons change, and encoder-row.js listens for it

**app.js updates:**
- Import `{ initJogWheel }` from `'./ui/jog-wheel.js'`
- Import `{ getOLEDElement }` from `'./ui/encoder-row.js'`
- After initEncoderRow: create a wrapper div for the encoder controls area, get the OLED ref, init the jog wheel
- In the toolbar button click handler (the existing `openSheet`/`closeSheet` logic), dispatch `mode-change` CustomEvent:
  - When `data-sheet="seq-sheet"` becomes active → dispatch `{ mode: 'drum' }`
  - When seq-sheet closes or another sheet opens → dispatch `{ mode: 'melodic' }`
- This keeps the coupling clean — app.js dispatches mode, encoder-row.js reacts

**Layout CSS adjustment (if not already handled):**
