<objective>
Build the rotary encoder UI component, OLED contextual display, and a row of 9 encoders wired to synth/FX parameters. This replaces the slider-based panels (SYN, FX bottom sheets) with Move-style encoder controls that sit directly in the main instrument layout.

Purpose: Encoders are the core interaction model for Ableton Move — parameters are adjusted by turning physical knobs, not dragging sliders in hidden panels. The OLED display provides contextual feedback only while interacting.

Output: Three new UI modules (encoder.js, oled-display.js, encoder-row.js), updated layout in index.html, and CSS for encoder/OLED styling.
</objective>

  <name>Task 1: Create rotary encoder component, OLED display, and encoder row module</name>
  <files>public/ui/encoder.js, public/ui/oled-display.js, public/ui/encoder-row.js</files>
  <action>
Create three new ES modules:

**public/ui/encoder.js** — `createEncoder(config)` factory function:
- config: `{ name, min, max, value, step, onChange }`
- Returns a DOM element: a circular div (40px diameter on mobile, 48px on desktop) with a dot indicator showing current position
- Visual: `#222` background, `#555` border (1px), white indicator dot at current angle. On interaction: border becomes `#888`, indicator dot becomes white
- Touch/pointer gesture: On pointerdown, track vertical drag distance (upward = increase, downward = decrease). Sensitivity: 200px full range. Use pointer capture for reliable tracking
- Calls `onChange(newValue)` during drag with clamped value
- Dispatches CustomEvents `encoder-start` and `encoder-end` on the element for OLED integration
- The indicator dot rotates from 7 o'clock (min) to 5 o'clock (max) — a 300-degree arc, matching real hardware encoder feel
- Store current value in element.dataset.value and expose via element.getValue() / element.setValue(v) methods attached to the DOM element

**public/ui/oled-display.js** — OLED contextual display:
- `createOLED()` returns a DOM element: a small rectangular display (full encoder-row width, 28px height)
- Visual: black background (#000), white monospace text, 0.7rem font size, centered. Border: 1px solid #111 (barely visible)
- `showOLED(displayEl, name, value)` — sets text content to `name  value` (name left, value right using flexbox justify-content: space-between), sets opacity to 1
- `hideOLED(displayEl)` — after 1500ms idle timeout, fade opacity to 0 over 300ms. Each call to showOLED resets the timeout
- When not active, the display is fully dark (opacity: 0) — "goes dark when not in use"
- Value formatting: numbers get 0-2 decimal places depending on step size; display shows unit-appropriate text (e.g., "Cutoff  2800 Hz", "Reverb  0.30")

**public/ui/encoder-row.js** — row of 9 encoders + OLED:
- `initEncoderRow(containerEl)` builds the encoder row and OLED display, appends to containerEl
- `setEncoderMapping(mappingArray)` swaps the parameter bindings (for Phase 7 Plan 03 contextual mapping)
- Default melodic mapping — 9 encoders in order:
  1. Filter Cutoff (50–10000, default 2800) → `filterFX.frequency.value`
  2. Filter Res (0.1–20, default 1) → `filterFX.Q.value`
  3. Reverb (0–1, default 0.3) → `reverb.wet.value`
  4. Delay (0–1, default 0) → `delay.wet.value`
  5. Attack (0.001–2, default 0.02) → `setSynthParam({ envelope: { attack: val } })`
  6. Release (0.01–3, default 0.4) → `setSynthParam({ envelope: { release: val } })`
  7. Distortion (0–1, default 0) → `distortion.distortion = val; distortion.wet.value = val > 0 ? 1 : 0`
  8. Vibrato (0–1, default 0) → `vibrato.depth.value = val; vibrato.wet.value = val > 0 ? 1 : 0`
  9. Volume (-40–0, default -6) → `masterVolume.volume.rampTo(val, 0.05)`
- Each encoder listens for `encoder-start` → calls `showOLED(oled, name, formattedValue)` and `encoder-end` → calls `hideOLED(oled)`
- During drag (onChange), update OLED with current value
- Layout: OLED display on top, 9 encoders in a flex row below with `justify-content: space-between`, `gap: 4px`
- Export `MELODIC_MAPPING` and `DRUM_MAPPING` arrays as named constants for Plan 03 to use
- DRUM_MAPPING: Kick Pitch, Kick Decay, Snare Tone, HH Decay, Reverb, Delay, BPM, Swing (placeholder), Volume — these are placeholder configs that Plan 03 will wire to actual drum params
