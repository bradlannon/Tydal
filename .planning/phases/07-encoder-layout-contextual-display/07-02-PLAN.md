<task type="auto">
  <name>Task 1: Create 16-step horizontal button row module</name>
  <files>public/ui/step-buttons.js</files>
  <action>
Create `public/ui/step-buttons.js` — a new ES module:

- `initStepButtons(containerEl)` builds and appends a horizontal row of 16 step buttons
- Layout: a single flex row, `gap: 3px`, with a slightly larger gap (8px) after every 4th button to create beat grouping markers
- Each button: 36px wide, 28px tall, border-radius 4px, background #181818, border: none. Touch-action: manipulation
- Active steps: background #fff (white), no glow
- Playhead step: background var(--move-glow-green) with subtle green glow `box-shadow: 0 0 6px rgba(0,255,90,0.4)`
- Active + playhead: green background with slightly stronger glow

**Behavior — Melodic mode (default):**
- Tapping a step toggles the note at that step for the currently selected note lane
- Import from melodic-sequencer.js: `getSelectedNote, hasNoteAtStep, getCurrentMelodicStep, getLanes`
- On tap, if a note is selected: toggle that note at the tapped step (dispatch appropriate event or call melodic-sequencer API)
- For now, step buttons show the FIRST lane's state (most recently tapped note). The step display listens for `melodic-update` events to refresh

**Behavior — Drum mode (when drum sequencer is active):**
- For Phase 7, step buttons only operate in melodic mode. Drum mode step editing remains in the DRM bottom sheet. Phase 8 will add track switching that routes step buttons to drum or melodic based on active track.

**Playhead tracking:**
- Listen for `sequencer-step` CustomEvent (dispatched by engine) and `melodic-update` CustomEvent
- On `sequencer-step`: highlight the current step column with playhead class
- On `melodic-update`: refresh which steps show as active (have notes)

**Page support:**
- Steps 0-15 shown simultaneously (no paging needed for 16 buttons in a row)
- Remove the page toggle concept for step buttons — all 16 steps visible at once (unlike the old 4x8 grid which could only show 8 at a time)

Export: `initStepButtons(containerEl)` — returns the step row element
