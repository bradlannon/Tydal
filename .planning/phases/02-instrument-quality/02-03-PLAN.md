<task type="auto">
  <name>Task 1: Add velocity to touch input and create MIDI handler</name>
  <files>input/touch.js, input/midi.js, app.js</files>
  <action>
Modify input/touch.js to measure velocity on pointerdown:
- Add module-level state: let lastPointerY = null, lastPointerTime = null
- In the pointerdown handler, compute speed from clientY delta / timestamp delta
- Map speed to velocity: 0 px/ms = 0.4 (minimum — never silent), ~3 px/ms = 1.0 (max). Use Math.min(1, Math.max(0.4, speed / 3))
- On first tap (lastPointerY === null), default to 0.6 (mid-range) — prevents Pitfall 4 from RESEARCH.md
- Pass velocity to noteOn(note, velocity)
- Update lastPointerY and lastPointerTime after each pointerdown
- Reset lastPointerY and lastPointerTime on 'grid-rebuild' event

Create input/midi.js:
- Export async function initMIDI(): returns boolean (true = MIDI available, false = not supported or denied)
- Check navigator.requestMIDIAccess exists. If not, log 'Web MIDI API not supported' and return false.
- Check window.isSecureContext. If not, log 'MIDI requires HTTPS' and return false.
- Try requestMIDIAccess(). On success, iterate inputs and attach onmidimessage = handleMIDIMessage
- Set onstatechange to re-attach handler on newly connected devices
- handleMIDIMessage(event): parse [status, noteNumber, velocityByte] from event.data
  - command = status & 0xf0
  - Note On (0x90 with velocityByte > 0): convert noteNumber to note string via Tone.Frequency(noteNumber, 'midi').toNote(), velocity = velocityByte / 127, call noteOn(note, velocity), call setPadActive(note, true)
  - Note Off (0x80, or 0x90 with velocityByte === 0): call noteOff(note), call setPadActive(note, false)
- Import noteOn, noteOff from instruments.js, setPadActive from pad-grid.js, ensureAudioStarted from audio-engine.js
- Call ensureAudioStarted() on first MIDI note-on (use a flag, not on every note)

Wire in app.js:
- Import { initMIDI } from './input/midi.js'
- Call initMIDI() after initTouch (no await needed — fire and forget, graceful on failure)

  <name>Task 2: Add LFO modulation nodes to effects chain</name>
  <files>engine/effects.js</files>
  <action>
Add LFO modulation to engine/effects.js (after the existing effects chain setup):
- Create vibrato: new Tone.Vibrato({ frequency: 5, depth: 0.1, wet: 0 }). Insert in signal chain BEFORE reverb: instrument -> vibrato -> reverb -> ... Chain change: connectInstrument must now connect synth to vibrato, and vibrato connects to reverb. Update connectInstrument accordingly.
- Create tremolo: new Tone.Tremolo({ frequency: 4, depth: 0.5, wet: 0 }).start(). Note: Tremolo must be started even when wet=0. Insert after vibrato: instrument -> vibrato -> tremolo -> reverb -> ...
- Create filterLFO: new Tone.LFO({ frequency: 2, min: 400, max: 4000, type: 'sine' }). Connect to filterFX.frequency. Do NOT start by default — start only when user enables it.
- Export vibrato, tremolo, filterLFO
- Export setLFO(target, params): convenience function to update LFO parameters. target is 'vibrato' | 'tremolo' | 'filterLFO'. params is an object like { frequency: 3, depth: 0.2, wet: 0.5 }. For filterLFO, params can include { min, max, frequency, type }. For vibrato/tremolo, use .set(params). For filterLFO start/stop, check if params includes enabled: true/false.

IMPORTANT: vibrato and tremolo default wet=0 (off). They are in the chain but inaudible until UI enables them.
IMPORTANT: Update connectInstrument to connect synth -> vibrato (first in chain) instead of directly to reverb. Update disconnectInstrument to disconnect from vibrato.
