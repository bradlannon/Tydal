Purpose: This is the visual foundation for the entire v1.1 redesign. Every subsequent phase builds on this black canvas where only illuminated elements exist.
Output: Updated styles.css with Move token system, stripped index.html toolbar, updated UI modules with label removal.

  <name>Task 1: Move CSS token system and black canvas overhaul</name>
  <files>public/styles.css, public/index.html</files>
  <action>
Replace the existing CSS custom property system with Move-inspired tokens. The goal is pure black (#000) everywhere with no visible borders, container backgrounds, or surface colors — only lit elements exist.

1. Replace `:root` variables:
   - `--move-black: #000` (body, all backgrounds)
   - `--move-surface: #111` (only for sheet backgrounds — the only non-black surface)
   - `--move-dim: #1a1a1a` (unlit pad/step background — barely visible)
   - `--move-glow-green: #00ff5a` (playhead — used in Plan 02)
   - `--move-glow-white: #ffffff` (active steps — used in Plan 02)
   - `--move-text-dim: #555` (only for contextual/transient text, not labels)
   - `--move-border: transparent` (all borders become invisible by default)
   - Remove `--bg-body`, `--bg-surface`, `--bg-surface-hover`, `--accent`, `--accent-glow`, `--text-primary`, `--text-secondary`, `--border-subtle` and all step-* vars
   - Keep `--header-height`, `--toolbar-height`, `--sheet-height` as-is

2. Update `body` background to `var(--move-black)` (#000).

3. Strip the header entirely:
   - In index.html, remove the `<header class="app-header">` element completely (the hardware has no title bar)
   - In styles.css, remove `.app-header` and `.app-title` rules

4. Make `.app-main` padding 0 (no gap around edges — hardware is edge-to-edge).

5. Update toolbar:
   - Background: `var(--move-black)` not `var(--bg-body)`
   - Border-top: `none` (was `1px solid var(--border-subtle)`)
   - `.toolbar-btn`: background `transparent`, border `none`, color `#444` (dim unlit), font-size `0.7rem`
   - `.toolbar-btn.active`: color `#fff`, background `transparent` (lit = white text, no colored background)
   - `.toolbar-btn--play`: same transparent style, `.playing` state gets color `var(--move-glow-green)`

6. In index.html toolbar:
   - Replace text button labels with single-character icons/symbols:
     - "Synth" button -> use the existing text but add `aria-label` and rely on CSS to make it minimal (actually keep short text like "SYN", "FX", "DRM", "MCR" as 3-char abbreviations since Move's physical buttons have tiny printed labels beneath — these are acceptable as they're tiny and dim)
   - Remove the `<span id="octave-display" class="octave-display">Oct 3</span>` text — octave buttons (+/-) remain but the "Oct 3" label goes
   - Remove the help button and help panel entirely (hardware has no help tooltip)
   - Remove the volume slider label (keep the range input)
   - Remove scale control labels (keep the selects but no "Scale key" / "Scale type" visible text)

7. Update octave buttons: background `transparent`, border `1px solid #333`, color `#555`. No octave display text.

8. Update range inputs: track background `#111`, thumb `#555` (dim), no glow shadow. Active/interacting thumb: `#fff`.

9. Bottom sheets: background `#000` (was `#111`), border-top-radius stays. Sheet title color `#444` (very dim). Sheet handle `#333`.

10. All `.panel-label`, `.panel-heading`, `.control-label`, `.bpm-control label`, `.seq-row-label`, `.lane-label`, `.page-label` — set `display: none`. These text labels must not be visible.

11. `.note-cell` base: background `var(--move-dim)` (#1a1a1a), border `none`. The `.cell-note` text (note names on pads) — set `display: none` (hardware pads have no text).

12. Update audio overlay: background `rgba(0, 0, 0, 0.96)`, accent line color `#333`.

13. Update `.grid-cell` border-radius from 6px to 4px (Move pads are slightly less rounded).

14. Remove all `box-shadow` glow effects from non-active states. Active states will be restyled in Plan 02.
