  <name>Task 1: Create pad grid, keyboard input, touch input, and help tooltip</name>
  <files>ui/pad-grid.js, input/keyboard.js, input/touch.js</files>
  <action>
Create the three interaction modules:

1. **ui/pad-grid.js** — Generates the 4x4 pad grid DOM:
   - Define the NOTE_MAP array: 16 entries, each with `{ note, key, padIndex }`. Use diatonic two-octave range C3-D5 for musical warmth:
     - Pad 1 (bottom-left): C3, key 'z'
     - Pad 2: D3, key 'x'
     - Pad 3: E3, key 'c'
     - Pad 4: F3, key 'v'
     - Pad 5: G3, key 'a'
     - Pad 6: A3, key 's'
     - Pad 7: B3, key 'd'
     - Pad 8: C4, key 'f'
     - Pad 9: D4, key 'q'
     - Pad 10: E4, key 'w'
     - Pad 11: F4, key 'e'
     - Pad 12: G4, key 'r'
     - Pad 13: A4, key '1'
     - Pad 14: B4, key '2'
     - Pad 15: C5, key '3'
     - Pad 16: D5, key '4'
   - Export NOTE_MAP so keyboard.js and touch.js can reference it.
   - Export `initPadGrid(containerEl)`:
     - Create a `div.pad-grid` element
     - CRITICAL MPC LAYOUT: Render rows in REVERSE order in the DOM. Top DOM row = pads 13-16, then 9-12, then 5-8, bottom DOM row = pads 1-4. This achieves bottom-left=lowest, top-right=highest visually.
     - Each pad is a `div.pad` with:
       - `data-note` attribute (e.g., "C3")
       - `data-key` attribute (e.g., "z")
       - Two spans inside: one for the keyboard shortcut letter (uppercase), one for the note name
       - `touch-action: manipulation` via CSS class (already in styles.css from Plan 01)
     - Append pad-grid to containerEl
   - Export `setPadActive(note, isActive)` — adds/removes `.active` class on the pad matching `data-note`. Used by both keyboard.js and touch.js for visual feedback.

2. **input/keyboard.js** (AUDIO-03 keyboard half):
   - Import `{ ensureAudioStarted }` from `'../engine/audio-engine.js'`
   - Import `{ noteOn, noteOff }` from `'../engine/instruments.js'`
   - Import `{ NOTE_MAP, setPadActive }` from `'../ui/pad-grid.js'`
   - Build KEY_TO_NOTE lookup from NOTE_MAP: `{ 'z': 'C3', 'x': 'D3', ... }`
   - `const heldKeys = new Set();`
   - Export `initKeyboard()`:
     - `document.addEventListener('keydown', async (e) => { ... })`:
       - `if (e.repeat) return;` — ignore auto-repeat
       - Look up `e.key.toLowerCase()` in KEY_TO_NOTE
       - If not found or already in heldKeys, return
       - Add to heldKeys
       - `await ensureAudioStarted();`
       - `noteOn(note);`
       - `setPadActive(note, true);`
     - `document.addEventListener('keyup', (e) => { ... })`:
       - Look up note from KEY_TO_NOTE
       - If not found, return
       - Remove from heldKeys
       - `noteOff(note);`
       - `setPadActive(note, false);`
   - ZERO setTimeout calls. All timing via Tone.now() inside noteOn/noteOff (AUDIO-05).

3. **input/touch.js** (AUDIO-03 touch half):
   - Import `{ ensureAudioStarted }` from `'../engine/audio-engine.js'`
   - Import `{ noteOn, noteOff }` from `'../engine/instruments.js'`
   - Import `{ setPadActive }` from `'../ui/pad-grid.js'`
   - `const touchedPads = new Map();` — pointerId -> note
   - Export `initTouch(padGridEl)`:
     - `padGridEl.addEventListener('pointerdown', async (e) => { ... }, { passive: false })`:
       - `e.preventDefault();` — MUST use `{ passive: false }` or Chrome ignores this (Pitfall 3)
       - `const pad = e.target.closest('[data-note]');`
       - If no pad, return
       - `const note = pad.dataset.note;`
       - `touchedPads.set(e.pointerId, note);`
       - `await ensureAudioStarted();`
       - `noteOn(note);`
       - `setPadActive(note, true);`
     - `padGridEl.addEventListener('pointerup', (e) => { ... })`:
       - Get note from touchedPads by pointerId
       - If no note, return
       - Delete from touchedPads
       - `noteOff(note);`
       - `setPadActive(note, false);`
     - `padGridEl.addEventListener('pointercancel', (e) => { ... })`:
       - IDENTICAL to pointerup handler — prevents stuck notes (Pitfall 6)
     - `padGridEl.addEventListener('pointerleave', (e) => { ... })`:
       - Same as pointerup — release note if finger slides off pad
   - Apply `touch-action: none` to the padGridEl via JS to prevent page scroll interference.
  </action>
