Implement tap tempo module per research Pattern 6:
- Constants: TAP_TIMEOUT_MS=2000, MIN_TAPS=2, MAX_TAPS=8

- Export recordTap and nothing else
- Import setBPM from '../engine/sequencer.js'

**ui/sequencer-ui.js:**
Export `initSequencerUI(containerEl)` that builds and wires the entire sequencer UI.

**DOM structure** (create programmatically, append to containerEl):
```
div.sequencer-section
  div.sequencer-transport
    button#seq-play.seq-btn  "Play" / "Stop" (toggles text)
    div.bpm-control
      label "BPM"
      input#bpm-slider type=range min=40 max=240 value=120 step=1
      span#bpm-display "120"
    button#tap-tempo.seq-btn "Tap"
  div.sequencer-grid
    (for each row in ROWS)
      div.seq-row[data-row="{row}"]
        span.seq-row-label "{Row}" (capitalize first letter)
        (for each step 0..15)
          div.seq-cell[data-row="{row}"][data-step="{step}"]
```

**Grid interaction:**
- Click/tap on .seq-cell toggles active state: toggle CSS class 'active', call setStep(row, step, !currentState)
- Use pointerdown event (not click) for faster mobile response
- Active cells get class 'active'

**Play/stop button:**
- On click: if not playing, call initTransport(currentBPM) then startSequencer(), change text to "Stop"
- If playing, call stopSequencer(), change text to "Play"
- initTransport must be called before first startSequencer to configure the Transport loop

**BPM slider:**
- On input: call setBPM(Number(e.target.value)), update bpm-display text
- Initialize at 120

**Tap tempo button:**
- On pointerdown: call recordTap() from tap-tempo.js
- After each tap, read getBPM() and update both bpm-slider.value and bpm-display text (since tap tempo changes BPM in the engine, the UI must reflect it)

**Cursor sync:**
- Listen for 'sequencer-step' CustomEvent on document
- On event: remove 'playing' class from all .seq-cell elements, then add 'playing' class to all cells in column e.detail.step (`.seq-cell[data-step="${step}"]`)
- When step is -1 (stopped), remove 'playing' from all cells

**Beat grouping visual hint:**
- Every 4th step boundary gets a subtle left border (steps 0, 4, 8, 12) via CSS class 'beat-start' added during grid creation
