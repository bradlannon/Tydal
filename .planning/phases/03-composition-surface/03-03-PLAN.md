Purpose: This completes the composition surface by letting users record melodic parts over drum loops. Notes are quantized to a selectable grid (1/4 through 1/32 note), overdub stacks multiple recording passes, and undo removes the last pass. This is the key "compose music" capability.

  <name>Task 1: Create recorder engine with quantized recording, overdub, and undo</name>
  <files>engine/recorder.js</files>
  <action>
Create `engine/recorder.js` implementing quantized loop recording with overdub and undo per research Pattern 7.

**State:**
- `let _isRecording = false`
- `let _quantization = '16n'` (default: snap to 16th notes)
- `const overdubStack = []` — array of Tone.Part objects, one per recording pass
- `let activePart = null` — the Part currently being recorded into

**Quantization control:**
- `setQuantization(subdiv)` — accepts '4n', '8n', '16n', '32n'. Store in _quantization. Export.
- `getQuantization()` — return _quantization. Export.

**Recording lifecycle:**
- `startRecording()` — guard: only works if Transport is running (import isPlaying from sequencer.js; if !isPlaying() log warning and return). Set _isRecording = true. Create a new Tone.Part with callback that plays back recorded events:
  ```
  activePart = new Tone.Part((time, event) => {
    // Import noteOn from instruments.js for playback
    noteOn(event.note, event.velocity);
    // Schedule noteOff after duration
    Tone.getTransport().scheduleOnce((t) => {
      noteOff(event.note);
    }, time + event.duration);
  }, []);
  ```
  Set activePart.loop = false (rely on Transport loop to replay — see research open question 4).
  Set activePart.start(0).
  Export this function.

- `stopRecording()` — set _isRecording = false. If activePart exists, push it onto overdubStack. Set activePart.loop = true, activePart.loopStart = 0, activePart.loopEnd = '1m'. Set activePart = null. Export.

  IMPORTANT: activePart.loop is set to true AFTER recording stops so the recorded part loops on subsequent Transport cycles. During recording it is false to avoid double-triggering.

- `recordNote(note, velocity, duration)` — guard: if !_isRecording || !activePart, return. Get the current Transport position as seconds: `const pos = Tone.getTransport().seconds`. Quantize: `const quantizedPos = Tone.getTransport().nextSubdivision(_quantization)`. Actually, use the simpler approach: add the event at '@{_quantization}' time. But since Part.add() needs a time offset within the Part, compute the quantized position:
  ```
  const transport = Tone.getTransport();
  // Get position quantized to nearest subdivision
  const rawPos = transport.seconds;
  const subdivSeconds = Tone.Time(_quantization).toSeconds();
  const quantizedPos = Math.round(rawPos / subdivSeconds) * subdivSeconds;
  // Wrap to 1 measure (transport.loopEnd in seconds)
  const measureLength = Tone.Time('1m').toSeconds();
  const wrappedPos = quantizedPos % measureLength;
  ```
  Then: `activePart.add(wrappedPos, { note, velocity, duration: Tone.Time(duration || '8n').toSeconds() })`.
  Export this function.

- `undoLastOverdub()` — pop last Part from overdubStack. Call .stop(), .dispose(). Return the new overdubStack.length. Export.

- `clearAllRecordings()` — iterate overdubStack, stop and dispose each. Clear array. If activePart, stop, dispose, set null. Set _isRecording = false. Export.

- `isRecording()` — return _isRecording. Export.
- `getOverdubCount()` — return overdubStack.length + (_isRecording ? 1 : 0). Export.

**Integration hook — modify instruments.js:**
Add a recording hook to noteOn in instruments.js. At the top of noteOn(), after the voice stealing logic, check if recorder is recording and call recordNote:
```javascript
import { isRecording, recordNote } from './recorder.js';
// Inside noteOn(), after voice steal + trackNoteOn:
if (isRecording()) {
  recordNote(note, velocity, '8n');
}
```
This is the cleanest integration point — all note sources (keyboard, touch, MIDI) flow through noteOn().
