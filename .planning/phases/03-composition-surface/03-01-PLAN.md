  <name>Task 1: Create drum synthesis engine with four pre-allocated voices</name>
  <files>engine/drums.js</files>
  <action>
Create `engine/drums.js` with four 808/909-style drum voices, all pre-allocated at module load.

**Drum bus routing:** Create a dedicated `Tone.Channel` as `drumBus` at 0 dB. Connect drumBus to `masterVolume` from `engine/effects.js`. All drum voices route through drumBus — they bypass the melodic effects chain (vibrato/tremolo/reverb/delay/distortion) because drum sounds need their own dedicated processing.

**Kick (DRUM-01):**
- `Tone.MembraneSynth` with pitchDecay:0.08, octaves:6, envelope {attack:0.001, decay:0.4, sustain:0, release:0.1}
- Layer with dedicated `Tone.Distortion` at distortion:0.4, wet:0.3
- Route: kick -> kickDist -> drumBus
- Export `triggerKick(time)` that calls `kick.triggerAttack('C1', time)` (C1 = deep 808-style sub)

**Snare (DRUM-02):**
- Noise component: `Tone.NoiseSynth` (white noise), envelope {attack:0.001, decay:0.15, sustain:0, release:0.05}, through `Tone.Filter` highpass at 1800Hz
- Tonal body: `Tone.Synth` (triangle oscillator), envelope {attack:0.001, decay:0.1, sustain:0, release:0.05}
- Both route to drumBus independently (mixed at equal level)
- Export `triggerSnare(time)` that fires both noise.triggerAttackRelease('16n', time) and tone.triggerAttack('E3', time) (E3 ~165Hz for body resonance; use triggerAttackRelease for the tone layer too, duration '16n')

**Hi-hat (DRUM-03):**
- Two `Tone.MetalSynth` instances: closedHat (decay:0.07, release:0.01) and openHat (decay:0.5, release:0.2)
- Both: frequency:400, harmonicity:5.1, modulationIndex:32, resonance:4000, octaves:1.5
- Both route to drumBus via a shared Tone.Gain node at -6dB (MetalSynth is loud)
- Export `triggerHihat(type, time)` where type is 'closed' or 'open'
- CHOKE behavior: when 'closed' fires, call openHat.triggerRelease(time) THEN schedule closedHat.triggerAttack at time+0.005 (5ms offset to avoid envelope overlap click). When 'open' fires, call closedHat.triggerRelease(time) then openHat.triggerAttack at time+0.005.

**Clap (DRUM-04):**
- `Tone.NoiseSynth` (white noise), envelope {attack:0.001, decay:0.06, sustain:0, release:0.02}
- Through `Tone.Filter` bandpass at frequency:1200, Q:0.5
- Through dedicated `Tone.Reverb` decay:0.8, wet:0.4
- Route clapReverb to drumBus
- Export `triggerClap(time)` that fires 4 staggered bursts: triggerAttackRelease('32n', time), +0.008, +0.016, +0.028 (irregular spacing for natural feel). The last burst uses '64n' for a shorter tail.

**DRUM-05 compliance:** All synths constructed at module top level. No `new Tone.*` inside trigger functions. Export `drumBus` for potential future per-drum mixing.

<task type="auto">
  <name>Task 2: Create step sequencer engine with Transport control and BPM</name>
  <files>engine/sequencer.js</files>
  <action>
Create `engine/sequencer.js` with a 16-step Tone.Sequence-based drum sequencer and Transport control.

**Transport setup:**
- `initTransport(bpm = 120)` — sets Transport bpm, loop:true, loopStart:0, loopEnd:'1m' (1 measure = 16 sixteenth notes in 4/4)
- `setBPM(bpm)` — clamp to 40-240, set transport.bpm.value
- `getBPM()` — return transport.bpm.value

**Grid state:**
- `const ROWS = ['kick', 'snare', 'hihat', 'clap']` (exported)
- `const NUM_STEPS = 16` (exported)
- Internal grid object: `{ kick: Array(16).fill(false), snare: ..., hihat: ..., clap: ... }`
- `setStep(row, step, active)` — set grid[row][step] = active (exported)
- `getStep(row, step)` — return grid[row][step] (exported)
- `getGrid()` — return reference to grid object (exported, for UI rendering)
- `getCurrentStep()` — return currentStep index (exported, for cursor)

**Sequence:**
- Create `Tone.Sequence` with callback `(time, step)`, array of step indices [0..15], subdivision '16n'
- sequence.loop = true
- In callback:
  1. Set currentStep = step
  2. If grid.kick[step] → triggerKick(time)
  3. If grid.snare[step] → triggerSnare(time)
  4. If grid.hihat[step] → triggerHihat('closed', time) (default to closed; TODO: UI can set open/closed per step later)
  5. If grid.clap[step] → triggerClap(time)
  6. Dispatch a custom event 'sequencer-step' with detail:{step} via Tone.getDraw().schedule() for visual cursor sync (CRITICAL: must use Draw, never update DOM directly from Sequence callback)
  7. FALLBACK: If Tone.getDraw is undefined, try Tone.Draw.schedule(). If neither exists, use requestAnimationFrame as last resort. Log a warning on first use of fallback.

**Transport controls:**
- `startSequencer()` — call ensureAudioStarted() from audio-engine.js, then transport.start(), then sequence.start(0). Export this.
- `stopSequencer()` — sequence.stop(), transport.stop(), reset currentStep to 0, dispatch 'sequencer-step' with step:-1 (cursor off). Export this.
- `isPlaying()` — return transport.state === 'started' (exported)

**Event interface:** The 'sequencer-step' CustomEvent is dispatched on `document` so any UI module can listen without importing the sequencer. Detail: `{ step: number }` where step is 0-15 during playback, -1 when stopped.

Import trigger functions from './drums.js'. Import ensureAudioStarted from './audio-engine.js'.
