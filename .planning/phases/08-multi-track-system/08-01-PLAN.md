**Create `public/engine/track-manager.js`** — the central track state manager:

1. Define TRACK_COLORS array: `['#e87a20', '#00b3f4', '#b44aff', '#00e676']` (orange, blue, purple, green) — 4 distinct track colors.

2. Define a `createMelodicTrack(id, color)` function that builds:
   - A new `Tone.PolySynth(Tone.Synth, { maxPolyphony: 8, options: { ... } })` using the same default subtractive synth params as instruments.js
   - A per-track effects chain via `createTrackEffectsChain()` (see effects.js changes below)
   - A 16-step melodic grid: `Array.from({ length: 16 }, () => new Set())`
   - Track object shape: `{ id, color, type: 'melodic', synth, effectsChain, grid, selectedNote: null, muted: false, volume: 0 }`
   - Wire synth into the track's effects chain: `synth.connect(effectsChain.input)`

3. Create 4 tracks at module level:
   - Track 0: `{ id: 0, color: TRACK_COLORS[0], type: 'drum', muted: false, volume: 0 }` — the drum track (uses existing drums.js engine, no separate synth)
   - Tracks 1-3: `createMelodicTrack(1, TRACK_COLORS[1])`, etc.

4. Module state: `let activeTrackId = 1` (default to first melodic track — this is what pads play on startup)

5. Exports:
   - `TRACK_COLORS` — the 4 color constants
   - `tracks` — the array of 4 track objects
   - `getActiveTrack()` — returns `tracks[activeTrackId]`
   - `setActiveTrack(id)` — sets `activeTrackId`, dispatches `CustomEvent('track-change', { detail: { trackId: id } })`
   - `getTrackById(id)` — returns `tracks[id]`
   - `getActiveTrackId()` — returns `activeTrackId`

**Modify `public/engine/effects.js`** — add `createTrackEffectsChain()` factory:

1. Keep ALL existing exports intact (vibrato, tremolo, reverb, delay, etc.) — these become the "Track 1 default" chain and are still referenced by encoder-row.js, macros.js, etc.

2. Add a new exported function `createTrackEffectsChain()` that creates an independent mini effects chain for a melodic track:
   ```javascript
   export function createTrackEffectsChain() {
     const trackReverb = new Tone.Reverb({ decay: 2.5, wet: 0.3 });
     const trackDelay = new Tone.FeedbackDelay({ delayTime: '8n', feedback: 0.35, wet: 0 });
     const trackChannel = new Tone.Channel({ volume: 0, pan: 0 });
     // Chain: input (reverb) -> delay -> channel -> masterVolume
     trackReverb.connect(trackDelay);
     trackDelay.connect(trackChannel);
     trackChannel.connect(masterVolume);
     return { input: trackReverb, reverb: trackReverb, delay: trackDelay, channel: trackChannel, ready: trackReverb.ready };
   }
   ```
   Each track gets reverb + delay (the "up to 2 effects" from MTRK-04). The channel node provides per-track volume/pan. All chains merge into the shared `masterVolume`.

3. The existing globals (vibrato, tremolo, reverb, delay, distortion, filterFX, channel) remain as they are — they will serve as the default chain for Track 1. The `connectInstrument`/`disconnectInstrument` helpers remain unchanged for backward compatibility.

**Modify `public/engine/melodic-sequencer.js`** — per-track melodic grids:

1. Import `tracks, getActiveTrack, getActiveTrackId` from `./track-manager.js`.

2. Replace the single `grid` array with a function that reads from the active track: the melodic grid now lives on each track object (`track.grid`). Remove the module-level `const grid = ...`.

3. The existing Tone.Sequence callback currently does:
   ```javascript
   for (const note of grid[step]) {
     activeSynth.triggerAttackRelease(note, '16n', time, 0.7);
   }
   ```
   Replace with: iterate ALL melodic tracks (tracks 1, 2, 3) and trigger each track's notes using that track's synth:
   ```javascript
   for (const track of tracks) {
     if (track.type !== 'melodic' || track.muted) continue;
     for (const note of track.grid[step]) {
       track.synth.triggerAttackRelease(note, '16n', time, 0.7);
     }
   }
   ```
   This ensures all 3 melodic tracks play simultaneously.

4. Update `toggleStep(step, note)` to operate on `getActiveTrack().grid[step]` instead of the old module-level grid. Only operates if active track is melodic.

5. Update `hasNoteAtStep(step, note)` to read from `getActiveTrack().grid[step]`.

6. Update `clearAllMelodic()` to clear only the active track's grid (or optionally all — use active track only).

7. Update `setSelectedNote(note)` to store on `getActiveTrack().selectedNote`.

8. Update `getSelectedNote()` to read from `getActiveTrack().selectedNote`.

9. Keep the `selectedNote` module var as a fallback for when track-manager isn't loaded, but prefer track state.

10. Remove the `import { activeSynth }` from instruments.js — no longer needed since each track has its own synth.

**Modify `public/engine/instruments.js`** — make noteOn/noteOff track-aware:

4. Keep `activeSynth` export and `switchInstrument()` — they still work but now the "active synth" concept means "the currently selected track's synth." Update `switchInstrument()` to swap the synth on the active track object and reconnect to that track's effects chain.

5. `getActiveSynth()` should return `getActiveTrack().synth` when a melodic track is active, falling back to `activeSynth`.
